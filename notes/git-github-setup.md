# Git & GitHub Initial Setup

## Objective

Configure Git and connect a local Linux repository with GitHub using SSH.

---

# Problem 1

## Error

```
Author identity unknown
```

### Cause

Git was not configured with my username and email.

### Solution

```bash
git config --global user.name "Your Name"

git config --global user.email "your@email.com"
```

Verify

```bash
git config --global --list
```

---

# Problem 2

## Error

```
failed to push some refs
(fetch first)
```

### Cause

The remote repository already had commits.

The local repository had a different history.

### Solution

Since the remote only contained an autogenerated README, I replaced it with my local repository.

```bash
git push --force -u origin main
```

---

# Problem 3

## Error

```
GH007:
Your push would publish a private email address
```

### Cause

GitHub Email Privacy was enabled.

GitHub blocked commits that exposed my private email.

### Solution 1 (Recommended)

Use GitHub No-Reply Email

OR

### Solution 2

Disable

```
Block command line pushes that expose my email
```

inside GitHub Email Settings.

---

# Final Push

```bash
git push --force -u origin main
```

Output

```
branch 'main' set up to track 'origin/main'
```

Repository successfully uploaded.

# Lessons Learned

Git stores my identity using:

git config --global

SSH authentication is different from Git identity.

GitHub Email Privacy can reject commits.

A repository can have different local and remote histories.

Force push should only be used when I intentionally want to replace the remote history.
